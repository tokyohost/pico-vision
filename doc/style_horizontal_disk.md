# 横向磁盘统计表盘说明

`style_horizontal_disk.py` 实现名为 `horizontal_disk` 的 LCD 表盘样式。该样式面向 320×240 横屏，在保留 CPU、内存和网络概览的同时，把主要显示空间用于磁盘总览和各分区统计。

## 启用方式

监控端通过以下参数选择该样式：

```text
--lcd-style horizontal_disk
```

也可以设置环境变量：

```text
PICO_MONITOR_LCD_STYLE=horizontal_disk
```

样式使用 `screen_2inch` 点阵字体，并声明 `landscape = True`，由渲染器自动切换为 320×240 横向显示。

## 表盘布局

```text
┌──────────── CPU ────────────┐ ┌──────────── STORAGE OVERALL ────────────┐
│ CPU                  占用率 │ │ 总已用/总容量                    占用率 │
│ 温度                        │ │ 总体容量进度条                         │
│ CPU 历史实心面积图          │ ├────────┬────────┬────────┤
└─────────────────────────────┘ │ 磁盘 1 │ 磁盘 2 │ 磁盘 3 │
┌──────────── MEM ────────────┐ ├────────┼────────┼────────┤
│ 占用率       容量进度条     │ │ 磁盘 4 │ 磁盘 5 │ 磁盘 6 │
│ 已用/总容量                 │ ├────────┼────────┼────────┤
└─────────────────────────────┘ │ 磁盘 7 │ 磁盘 8 │ 磁盘 9 │
┌──────────── NET ────────────┐ └────────┴────────┴────────┘
│ NET                 P延迟ms │
│ ↑UP 上传速率                │
│ 上传历史实心面积图          │
│ ↓DN 下载速率                │
│ 下载历史实心面积图          │
└─────────────────────────────┘
┌──────── 时间 ─────── UPTIME ─────────────── PWR ────────┐
└───────────────────────────────────────────────────────────┘
```

左侧区域宽 100 像素，依次显示 CPU、内存和网络；右侧区域宽 212 像素，显示磁盘总览及三列三行的磁盘卡片；底部为全宽状态栏。

## CPU 区域

CPU 区域位于左上角，显示以下内容：

- `CPU` 标题和当前 CPU 温度。
- 右侧使用二倍字号显示当前 CPU 占用率。
- 下方显示 CPU 历史占用率实心面积图，纵轴固定为 0%～100%。
- 面积图和百分比使用当前 CPU 占用率对应的状态颜色。

CPU 和内存占用率颜色规则：

| 数值范围 | 颜色 | 含义 |
| --- | --- | --- |
| 小于 50% | 绿色 | 正常 |
| 50%～79% | 黄色 | 较高 |
| 大于等于 80% | 红色 | 危险 |

## 内存区域

内存区域显示：

- `MEM` 标题。
- 当前内存占用百分比及同色进度条。
- 已用容量和总容量，例如 `51.2/63.9G`。

已用容量与总容量单位相同时，左侧重复单位会被省略。容量按照 B、K、M、G、T 自动换算，其中 G 和 T 保留一位小数。

## 网络区域

网络区域标题缩写为 `NET`，包含延迟、上传和下载信息：

- 延迟显示在标题右侧，例如 `P23ms`；无法取得延迟时显示 `ERR`。
- `↑UP` 表示上传，具体速率使用蓝色，与上传实心面积图一致。
- `↓DN` 表示下载，具体速率使用绿色，与下载实心面积图一致。
- 网络速率根据监控端配置显示为 `B/s`、`KB/s`、`MB/s`、`GB/s`，或者比特速率模式。
- 上传和下载趋势图根据各自历史数据的最大值自动缩放。

网络延迟颜色规则：

| 延迟范围 | 颜色 | 含义 |
| --- | --- | --- |
| 小于 50ms | 绿色 | 良好 |
| 50ms～99ms | 黄色 | 一般 |
| 大于等于 100ms | 红色 | 较高 |
| 无数据 | 红色 | 检测异常 |

## 磁盘总览

右上角 `STORAGE OVERALL` 区域显示全部磁盘的总体统计：

- 已用总容量与总容量。
- 总体磁盘占用百分比。
- 总体容量进度条。

总览边框和标题保持黄色，百分比及进度条按照磁盘占用率动态变色。

## 磁盘卡片

表盘最多显示 9 个磁盘，按三列三行排列。每张卡片包含：

- 盘符或磁盘名称，例如 `C:\`。
- 当前磁盘温度；无数据时显示灰色 `--C`。
- 已用容量与总容量，使用共享单位并压缩到最多 8 个字符。
- 当前占用百分比和容量进度条。

每个磁盘的边框、盘符、占用百分比和进度条使用相同的占用率颜色：

| 数值范围 | 颜色 | 含义 |
| --- | --- | --- |
| 小于 50% | 绿色 | 空间充足 |
| 50%～89% | 黄色 | 空间逐渐紧张 |
| 大于等于 90% | 红色 | 空间不足 |

## 温度颜色

CPU 温度和所有磁盘温度统一使用以下分级：

| 温度范围 | 颜色 | 含义 |
| --- | --- | --- |
| 小于 50°C | 绿色 | 正常 |
| 50°C～69°C | 黄色 | 偏高 |
| 大于等于 70°C | 红色 | 过高 |
| 无数据 | 灰色 | 未检测到温度 |

温度颜色独立于占用率颜色。例如磁盘空间充足但温度过高时，磁盘边框仍为绿色，温度文本会显示为红色。

## 底部状态栏

底部状态栏从左到右显示：

- 当前时间，格式为 `HH:MM:SS`。
- `UPTIME` 与系统运行时长，格式为 `天数D HH:MM:SS`。
- `PWR` 与当前功耗，单位为 W；无功耗数据时显示 `--W`。

## 动态刷新区域

为减少 Pico LCD 刷新数据量，表盘被拆分为独立脏矩形：

| 区域键 | 坐标与尺寸 | 内容 |
| --- | --- | --- |
| `cpu` | `(2, 2, 100, 69)` | CPU 信息和趋势图 |
| `memory` | `(2, 75, 100, 48)` | 内存信息 |
| `network` | `(2, 127, 100, 82)` | 网络信息和趋势图 |
| `storage_summary` | `(106, 2, 212, 43)` | 磁盘总体统计 |
| `disk_row_0` | `(106, 49, 212, 48)` | 第一行磁盘卡片 |
| `disk_row_1` | `(106, 101, 212, 48)` | 第二行磁盘卡片 |
| `disk_row_2` | `(106, 153, 212, 48)` | 第三行磁盘卡片 |
| `footer` | `(2, 213, 316, 25)` | 底部状态栏 |

首次绘制时，渲染器按当前条带与各区域的交集绘制完整表盘；后续刷新只重绘发生变化的区域。
