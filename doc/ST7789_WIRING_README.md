# Raspberry Pi Pico 与 ST7789 LCD 接线说明

本文档适用于项目 `picoRP2040` 当前的 ST7789 配置。屏幕使用 SPI0 单向传输，
不需要连接 MISO。接线前请先断开 Pico 的 USB 电源。

## 当前引脚配置

```python
PIN_SCK = 6
PIN_MOSI = 7
PIN_CS = 8
PIN_DC = 14
PIN_RST = 15
PIN_BL = 26
X_OFFSET = 0
Y_OFFSET = 0
```

配置中的数字是 GPIO 编号，不是开发板实体针脚编号。

## 接线对照表

| ST7789 模块标识 | 功能 | Pico GPIO | Pico 实体针脚 | 说明 |
| --- | --- | ---: | ---: | --- |
| `GND` | 电源地 | `GND` | 例如 8、13、18、23、28、38 | Pico 与屏幕必须共地 |
| `VCC` | 屏幕电源 | `3V3(OUT)` | 36 | 推荐使用 3.3V 供电 |
| `SCL`、`SCK` 或 `CLK` | SPI 时钟 | `GP6` | 9 | 对应 `PIN_SCK = 6` |
| `SDA`、`MOSI` 或 `DIN` | SPI 数据 | `GP7` | 10 | 对应 `PIN_MOSI = 7`，不是 I²C SDA |
| `CS` | 片选 | `GP8` | 11 | 对应 `PIN_CS = 8` |
| `DC`、`A0` 或 `RS` | 数据/命令选择 | `GP14` | 19 | 对应 `PIN_DC = 14` |
| `RES`、`RST` 或 `RESET` | 硬件复位 | `GP15` | 20 | 对应 `PIN_RST = 15` |
| `BL`、`BLK` 或 `LED` | 背光控制 | `GP26` | 31 | 对应 `PIN_BL = 26`，高电平开启 |

> 不同厂家的屏幕丝印可能不同。`SDA` 在这类 SPI 屏幕上通常表示 MOSI 数据线，
> 不能接到 Pico 的 I²C SDA。

## 简易接线图

```text
Raspberry Pi Pico                         ST7789 LCD

3V3(OUT)，实体针脚 36  ----------------> VCC
GND，实体针脚 38       ----------------> GND
GP6，实体针脚 9        ----------------> SCL / SCK / CLK
GP7，实体针脚 10       ----------------> SDA / MOSI / DIN
GP8，实体针脚 11       ----------------> CS
GP14，实体针脚 19      ----------------> DC / A0 / RS
GP15，实体针脚 20      ----------------> RES / RST / RESET
GP26，实体针脚 31      ----------------> BL / BLK / LED
```

## 接线步骤

1. 断开 Pico 的 USB 电源。
2. 先连接 `GND` 和 `VCC`，确认屏幕模块的供电要求。
3. 连接 `SCK`、`MOSI`、`CS`、`DC` 和 `RST`。
4. 将背光引脚连接到 `GP26`。
5. 再连接 USB 并启动 Pico 程序。

## 注意事项

- Pico GPIO 的逻辑电平是 3.3V，请勿向 GPIO 输入 5V。
- 推荐屏幕使用 Pico 的 `3V3(OUT)` 供电。只有模块说明明确支持 5V 输入时，
  才能考虑使用 `VBUS`，但其逻辑信号仍必须是 3.3V。
- 当前 SPI 时钟为 62.5 MHz。杜邦线应尽量短，建议控制在 15cm 以内，并确保接触可靠。
- `X_OFFSET` 和 `Y_OFFSET` 是显存显示偏移，不对应任何接线。
- 当前驱动没有读取屏幕数据，因此 ST7789 模块的 `MISO` 或 `SDO` 可以不接。
- 如果模块没有 `CS` 引脚，应先确认它是否已在模块内部固定接地，不要随意改接其他脚。

## 常见问题

### 屏幕完全不亮

检查 `VCC`、`GND` 和 `BL/BLK`。当前程序使用 `GP26` 控制背光，初始化后输出高电平。

### 背光亮但没有图像

重点检查 `SCK`、`MOSI`、`CS`、`DC` 和 `RST` 是否接反，并确认 Pico 正在运行
`picoRP2040/main.py`。

### 图像闪烁或偶尔花屏

缩短连接线并重新插紧。如果问题仍然存在，可临时降低 `lcd.py` 中 SPI 的
`baudrate=62_500_000`，例如改为 `20_000_000` 进行排查。

### 图像整体偏移或被裁切

这通常不是接线问题。不同尺寸的 ST7789 面板可能需要调整 `config.py` 中的
`X_OFFSET`、`Y_OFFSET`、`WIDTH` 或 `HEIGHT`。

### 颜色或方向不正确

颜色反转、横竖方向和旋转由 ST7789 初始化命令及 `MADCTL` 配置决定，与 SPI 接线
通常无关。可先检查 Monitor 的屏幕旋转配置，再调整驱动方向参数。
